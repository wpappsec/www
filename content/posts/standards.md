## WPAPPSEC Standards

We have borrowed the A, AA and AAA category format from the WACG project. For example, for logins, forcing strong passwords is A standard, forcing MFA is AA standard and allow listing IP addresses is AAA.

For each point you have either Raised (members), Accepted (application) or Met (application) the criteria.

For each point there will be advice for differing roles. 

Currently there are no standards, but we are working on a draft for these..

For example designers working on a login sign up experiences they should ask to design with MFA in mind and provide UI to handle forcing storing passwords and UI for denied IP addresses. 

Each role has a set of requirements they should be raising with applications they work on.

An application can be compliant if all points in a category have been met or not met but the risk has been accepted. An application cannot be compliant if a point has not been met and has also not been accepted. 

Assuming an application has met AA and has met or accepted all AAA issues an application to can claim to be WPAPPSEC AA met or WPAPPSEC AAA Accepted depending on what they wish to divulge.

A site can be WPAPPSEC AA compliant if it has  either met or formally accepted the risk in all points in the category.
